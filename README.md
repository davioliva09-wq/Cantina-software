Requisitos Funcionais (RF)

RF01 - O sistema deve permitir o cadastro de usuários com os seguintes dados: nome, número de matrícula, telefone, CPF, e-mail e senha.

RF02 - O sistema deve permitir o login dos usuários utilizando e-mail e senha.

RF03 - O sistema deve permitir aos usuários visualizar os produtos disponíveis e identificar visualmente sua disponibilidade em estoque.

RF04 - O sistema deve permitir aos usuários realizar pedidos de produtos.

RF05 - O sistema deve permitir aos usuários visualizar o valor total do pedido.

RF06 - O sistema deve registrar os pedidos realizados pelos usuários.

RF07 - O sistema deve permitir aos usuários consultar o histórico de pedidos.

RF08 - O sistema deve permitir que funcionários visualizem todos os pedidos realizados.

RF09 - O sistema deve permitir que funcionários alterem o status dos pedidos (Recebido, Em preparo, Pronto para retirada, Entregue e Cancelado).

RF10 - O sistema deve permitir que funcionários cadastrem novos produtos.

RF11 - O sistema deve permitir que funcionários editem produtos cadastrados.

RF12 - O sistema deve permitir que funcionários removam produtos cadastrados.

RF13 - O sistema deve permitir o controle de estoque dos produtos.

RF14 - O sistema deve informar quando um produto estiver indisponível.

RF15 - O sistema deve permitir a pesquisa de produtos por nome.

RF16 - O sistema deve permitir que usuários visualizem informações dos produtos, incluindo nome, preço, descrição e disponibilidade.

RF17 - O sistema deve permitir que funcionários visualizem relatórios de vendas.

RF18 - O sistema deve permitir que usuários recuperem sua senha.

RF19 - O sistema deve permitir que usuários alterem seus dados cadastrais.

RF20 - O sistema deve permitir que usuários realizem pagamentos online de seus pedidos.

RF21 - O sistema deve validar o pagamento antes de confirmar o pedido.

RF22 - O sistema deve gerar um comprovante de pagamento para o usuário.

RF23 - O sistema deve registrar a data e o horário do pagamento.

RF24 - O sistema deve permitir que funcionários visualizem a situação de pagamento dos pedidos.

RF25 - O sistema deve informar ao usuário quando o pagamento for aprovado ou recusado.

RF26 - O sistema deve gerar um código identificador único para cada pedido.

Requisitos Não Funcionais (RNF)

RNF01 - O sistema deve ser acessível por computadores e dispositivos móveis.

RNF02 - O sistema deve armazenar as senhas dos usuários de forma criptografada.

RNF03 - O sistema deve possuir interface simples e intuitiva.

RNF04 - O sistema deve responder às solicitações dos usuários em até 3 segundos.

RNF05 - O sistema deve utilizar banco de dados para armazenamento persistente das informações.

RNF06 - O sistema deve garantir a integridade dos dados cadastrados.

RNF07 - O sistema deve possuir autenticação para acesso às funcionalidades restritas.

RNF08 - O sistema deve manter registro das operações realizadas pelos funcionários.

RNF09 - O sistema deve estar disponível durante o horário de funcionamento da cantina.

RNF10 - O sistema deve seguir boas práticas de segurança para proteção dos dados dos usuários.
